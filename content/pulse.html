Pulse · Dashboard

Meet Pulse — a dashboard for RunMQ.

Pulse is the official web-based monitoring and management dashboard for RunMQ. Watch your queues live, inspect dead-letter messages, toggle retries and DLQ routing, and reprocess failed jobs — all from a clean, minimal UI with full dark-mode support.

Get started → ★ GitHub

What you get

◉

Real-time queue monitoring

Live message counts, consumer stats, and queue states with configurable auto-refresh.

≡

Queue detail view

Per-queue metrics: total, ready, unacknowledged, retry, and DLQ message counts.

↻

Retry management

Enable or disable automatic retries and tune retry delays per queue, without redeploying.

⌫

Dead letter queue

Toggle DLQ routing, inspect messages with full payload and headers, reprocess or clear.

⌕

Message inspector

Browse retry and DLQ messages with expandable JSON payloads, headers, and routing keys.

◐

Dark & light mode

Clean, minimal UI inspired by instatus.com — subtle borders, no heavy glassmorphism.

Getting started

Prerequisites

  • Docker and Docker Compose
  • A running RabbitMQ instance with the Management & Shovel plugins enabled

1. Clone the repository

git clone https://github.com/runmq/pulse.git
cd pulse

2. Point Pulse at your RabbitMQ

Update the environment variables in docker-compose.yml:

RABBITMQ_HOST: your-rabbitmq-host
RABBITMQ_PORT: 15672
RABBITMQ_MANAGEMENT_USER: your-user
RABBITMQ_MANAGEMENT_PASSWORD: your-password
JWT_SECRET: your-secret-key

3. Start the dashboard

docker compose up --build

4. Create an admin user

docker compose exec backend node scripts/create-user.js admin yourPassword admin

5. Open the UI

Visit http://localhost:3000 and sign in to start managing your queues.

Authentication. Pulse uses JWT with a 24-hour token expiration. Credentials are stored in backend/config/users.json with bcrypt-hashed passwords — no database required.

Development

Prefer to run without Docker? Start the backend and frontend separately:

# Backend (NestJS, port 3001)
cd backend && npm install && npm run start:dev

# Frontend (Next.js 14, port 3000)
cd frontend && npm install && npm run dev

How it works

Pulse talks to RabbitMQ's Management API directly — no extra database, no agents to install. RunMQ-managed queues are detected by their dead-letter exchange (_runmq_dead_letter_router), and toggles for retries and DLQ routing are persisted as RabbitMQ policies and bindings, so configuration travels with the broker itself.

DLQ reprocessing uses RabbitMQ shovels, available as one-time runs or on a cron schedule.
